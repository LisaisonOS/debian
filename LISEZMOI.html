LiaisonOS — Communications d'urgence pour radioamateurs

EmComm-Tools est une ISO Linux Debian 13 (Trixie) live et installable conçue pour les communications numériques d'urgence en radioamateur. Elle comprend une suite complète de logiciels de modes numériques préconfigurés, des cartes hors ligne et un tableau de bord web pour une utilisation en un clic. Deux modes de déploiement : USB Live Run (démarrer n'importe quel PC depuis une clé USB) ou installation sur disque dur (installation permanente avec Calamares). Interface bilingue anglais/français. Créé par Sylvain Deguire (VA2OPS), concept original de Gaston Gonzalez (KT7RUN). Version 2.2.3 — Avril 2026 ISO Debian 13 (Trixie) Live/Installable

⚠️ Avis de développement continu

LiaisonOS est développé selon un modèle de publication continue et agile — les mises à jour sont publiées fréquemment au fur et à mesure des fonctionnalités et correctifs, plutôt que d'attendre une grande version "finale". Chaque version est pleinement fonctionnelle, mais des changements récents peuvent occasionnellement introduire des problèmes inattendus dans d'autres domaines.

Démarrez et testez toujours depuis USB en premier. Une installation permanente vous offre la meilleure expérience, mais installer un logiciel qui n'a pas été testé pour votre usage et votre matériel spécifique peut entraîner des problèmes de fonctionnement. Tester depuis USB avant de procéder à une installation permanente est toujours l'approche recommandée.

Qu'est-ce qu'LiaisonOS?

LiaisonOS est une ISO Linux Debian 13 (Trixie) live et installable conçue pour les communications numériques d'urgence en radioamateur. Elle comprend une suite complète de logiciels de modes numériques préconfigurés, des cartes hors ligne et un tableau de bord web pour une utilisation en un clic. Deux modes de déploiement : USB Live Run (démarrer n'importe quel PC depuis une clé USB) ou installation sur disque dur (installation permanente avec Calamares). Interface bilingue anglais/français. Créé par Sylvain Deguire (VA2OPS).

Préinstallé : VARA, VarAC et Wine

LiaisonOS est la seule distribution Linux radioamateur qui livre VARA et VarAC prêts à l'emploi. Pas de configuration Wine manuelle, pas de téléchargement d'installateurs, pas de dépannage de DLL.

  • Wine 32 bits — Préfixe .wine32 préconfiguré, prêt à lancer les applications radioamateurs Windows
  • VARA HF et VARA FM — Modems haute performance préinstallés (inclus avec la permission de EA5HVK)
  • VarAC — Client de chat HF pour VARA, préinstallé (accord de distribution limitée avec 4Z1AC — licence présentée au premier lancement)
  • ISO Complète vs Lite — Le système de compilation offre les deux : l'ISO Complète inclut Wine/VARA/VarAC, l'ISO Lite les omet pour un téléchargement plus léger
  • Zéro configuration — VARA et VarAC se lancent depuis le Tableau de bord comme n'importe quelle application Linux native

Persistance (introduite en 1.5.2)

Lorsque vous démarrez LiaisonOS depuis une clé USB, tous vos changements — configurations radio, paramètres d'indicatif, identifiants Winlink, scripts personnalisés, préférences du bureau — sont normalement perdus à l'arrêt. Vous repartez à zéro à chaque fois.

La persistance change tout cela. Votre environnement utilisateur complet est sauvegardé sur une partition dédiée de la clé USB (EMCOMM-DATA). Au prochain démarrage, tout est exactement comme vous l'avez laissé : vos profils radio, votre boîte aux lettres Pat Winlink, vos paramètres JS8Call, vos signets — tout.

Trois façons d'utiliser la persistance

1. Démarrage USB en mode live (cas d'utilisation principal)

Démarrez n'importe quel PC depuis votre clé USB et reprenez exactement où vous en étiez. Configurez votre station une seule fois, puis transportez-la dans votre poche. Branchez-la sur n'importe quel ordinateur — votre station EmComm complète est prête en quelques minutes.

2. Sauvegarder votre installation permanente

Même si vous utilisez LiaisonOS installé sur un PC dédié, vous pouvez exporter toute votre configuration sur une clé USB. Si votre système principal tombe en panne pendant une urgence, démarrez depuis la clé USB sur n'importe quel ordinateur disponible — vos paramètres, identifiants et profils radio sont tous là.

3. Migrer entre appareils

Vous configurez un nouveau laptop ou remplacez un Toughpad endommagé? Au lieu de tout reconfigurer à partir de zéro, exportez vos données de persistance depuis l'ancien appareil et importez-les sur le nouveau. Toute la configuration de votre station se déplace avec vous — indicatif, profils radio, configuration Winlink, paramètres des modes numériques, tout se transfère en une seule étape.

Pourquoi c'est important pour les communications d'urgence

En situation d'urgence réelle, vous n'avez pas le temps de reconfigurer vos logiciels. La persistance signifie que votre clé USB devient une station EmComm complète et portable qui fonctionne sur n'importe quel matériel disponible. Vous perdez votre laptop? Empruntez n'importe quel PC, branchez votre clé USB, et vous êtes de retour en ondes avec toute votre configuration.

Important : Choisir la bonne méthode d'installation

Avant de continuer, comprenez quand utiliser le démarrage USB versus l'installation permanente.

Installation permanente (recommandée pour les opérations)

Un PC ou portable dédié avec LiaisonOS installé sur le disque interne est toujours la meilleure solution pour un travail sérieux de communications d'urgence.

  • Fiabilité maximale — Pas de clé USB à perdre ou oublier
  • Performance maximale — Vitesse native du disque, aucun goulot USB
  • Persistance intégrée — Toutes les configurations sauvegardées automatiquement
  • Prêt quand nécessaire — Allumez et opérez immédiatement
  • Déploiements professionnels — Hôpitaux de campagne, centres d'urgence, refuges

Utilisez l'installation permanente quand : - Vous opérez une station EmComm fixe - Vous faites fonctionner des services BBS ou passerelle 24/7 - Vous supportez des opérations d'urgence critiques - La fiabilité maximale est essentielle

Démarrage USB avec Ventoy (recommandé pour tests et sauvegarde)

Le démarrage USB est idéal pour les tests, la formation, les démonstrations et comme option de déploiement de secours.

  • Tests rapides — Testez de nouvelles ISO sans toucher à votre système principal
  • Portabilité — Transportez LiaisonOS dans votre poche
  • Multi-démarrage — Gardez plusieurs versions ISO sur une seule clé
  • Sauvegarde d'urgence — Démarrez n'importe quel PC disponible en situation de crise
  • Formation — Laissez les étudiants pratiquer sans changements permanents

Utilisez le démarrage USB quand : - Vous testez de nouvelles versions ISO avant déploiement - Vous formez de nouveaux opérateurs - Vous démontrez LiaisonOS lors d'événements - Option de secours si le système principal échoue - Conversion temporaire de n'importe quel PC en station EmComm

La vitesse USB compte : Utilisez une clé USB 3.x pour le démarrage Live. Les clés USB 2.0 causent des temps de démarrage, de lancement d'applications et de sauvegarde/restauration de persistance nettement plus lents. Une clé USB 3.0+ de qualité sur un port USB 3.0 fait une différence significative dans la réactivité globale.

Clé USB standard (méthode rapide)

La façon la plus simple de créer une clé USB démarrable est d'écrire l'ISO directement sur le disque. C'est la méthode recommandée pour la plupart des utilisateurs qui veulent simplement démarrer ou installer LiaisonOS.

Le seul logiciel dont vous avez besoin est un créateur USB démarrable. Voici les options les plus fiables par plateforme :

Windows

  • Rufus — Rapide, fiable, open-source. Le standard de référence pour Windows. rufus.ie
  • balenaEtcher — Processus simple en 3 étapes, valide les écritures, multiplateforme. etcher.balena.io
  • Ventoy — Multi-boot capable, mettez plusieurs ISO sur une USB. ventoy.net

macOS

  • balenaEtcher — Meilleur choix pour Mac, simple, fiable, application native. etcher.balena.io
  • UNetbootin — Multiplateforme, fonctionne bien pour les ISO Linux. unetbootin.github.io
  • dd (Terminal) — Intégré, puissant mais requiert de la prudence. sudo dd if=image.iso of=/dev/diskN bs=4M

Linux

  • balenaEtcher — Même interface simple que les autres plateformes. AppImage disponible.
  • Ventoy — Installez une fois, puis copiez simplement les fichiers ISO. ventoy.net
  • GNOME Disques — Intégré à la plupart des bureaux GNOME, fonction « Restaurer l'image disque ». Pré-installé.
  • dd — Outil Unix classique, rapide et fiable. sudo dd if=image.iso of=/dev/sdX bs=4M status=progress

Recommandation

Pour débutants : balenaEtcher — Fonctionne sur toutes les plateformes, impossible d'écraser accidentellement votre disque dur, valide l'écriture.

Pour utilisateurs avancés : Ventoy — Installez-le une fois sur votre clé USB, puis copiez simplement les fichiers ISO dessus. Vous pouvez avoir LiaisonOS, un installateur Windows et des outils de récupération sur une même USB. Voir le guide Ventoy détaillé ci-dessous.

Pourquoi Ventoy? La stratégie USB recommandée

Les méthodes traditionnelles de démarrage USB (dd, Rufus, balenaEtcher) écrivent l'ISO directement sur le disque, ce qui présente des limitations importantes.

Traditionnel vs Ventoy : - Une ISO par clé → Plusieurs ISO sur la même clé - Reflash complet pour les mises à jour → Simple copie/suppression de fichiers ISO - Tout l'espace utilisé pour le démarrage → Partition de données dédiée disponible - Données perdues à chaque reflash → La partition de données survit aux mises à jour - Processus complexe à chaque fois → Simple copie de fichier

L'avantage Ventoy

Ventoy transforme votre clé USB en gestionnaire de démarrage qui peut lancer n'importe quel fichier ISO simplement en le copiant sur le disque. C'est révolutionnaire pour les utilisateurs d'LiaisonOS :

  1. Testez de nouvelles versions instantanément — Téléchargez une nouvelle ISO, copiez-la sur USB, redémarrez. Pas de reflash, aucune perte de données.
  2. Gardez plusieurs versions — Maintenez la version stable aux côtés des versions bêta pour les tests.
  3. Séparez le démarrage des données — Vos cartes, fichiers ZIM et configurations résident sur une partition séparée qui survit aux mises à jour ISO.
  4. Flexibilité d'urgence — En situation d'urgence réelle, démarrez l'ISO stable éprouvée. Pour les tests, essayez la dernière version.

Aperçu de la stratégie de partitionnement

Ce guide crée une clé USB optimisée pour LiaisonOS avec la disposition suivante :

┌─────────────────────────────────────────────────────────────────────┐
│                  Disposition clé USB 64 Go                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────┐  ┌──────┐  ┌────────────────────────────────┐  │
│  │  EMCOMM-ISO     │  │ EFI  │  │        EMCOMM-DATA             │  │
│  │    ~12 Go       │  │ 32Mo │  │          ~46 Go                │  │
│  │                 │  │      │  │                                │  │
│  │  • Fichiers ISO │  │ Sys  │  │  • Cartes (.mbtiles)           │  │
│  │  • Ventoy       │  │ boot │  │  • Wikipédia (.zim)            │  │
│  │                 │  │      │  │  • Documents                   │  │
│  │  exFAT          │  │FAT16 │  │  • Sauvegardes config          │  │
│  └─────────────────┘  └──────┘  └────────────────────────────────┘  │
│       Partition 1      Part 2           Partition 3                 │
└─────────────────────────────────────────────────────────────────────┘
  • Partition 1 — EMCOMM-ISO, ~12 Go, exFAT — ISO amorçables via Ventoy
  • Partition 2 — (EFI), 32 Mo, FAT16 — Système de démarrage (automatique)
  • Partition 3 — EMCOMM-DATA, ~46 Go, ext4/exFAT — Cartes, fichiers ZIM, données hors ligne

Pourquoi cette disposition?

  • Partition ISO dimensionnée pour plusieurs versions — 12 Go contient 3-4 ISO LiaisonOS
  • Partition de données maximisée — Les cartes et fichiers Wikipédia peuvent atteindre plusieurs Go chacun
  • Compatible multiplateforme — exFAT sur la partition de démarrage fonctionne avec Windows/Mac/Linux
  • Données optimisées Linux — ext4 offre une meilleure fiabilité (ou exFAT si accès Windows nécessaire)

Prérequis

  • Clé USB 64 Go (ou plus)
  • Fichier ISO LiaisonOS
  • Accès administrateur/root

AVERTISSEMENT : Ce processus effacera toutes les données sur la clé USB. Sauvegardez vos fichiers importants d'abord!

Étape 1 : Télécharger Ventoy

Téléchargez la dernière version de Ventoy : - Site officiel : https://ventoy.net - GitHub : https://github.com/ventoy/Ventoy/releases

Téléchargements : - Windows : ventoy-x.x.xx-windows.zip - Linux : ventoy-x.x.xx-linux.tar.gz - macOS : Voir la section macOS ci-dessous

Étape 2 : Identifier votre clé USB

CRITIQUE : Identifier le bon disque est essentiel. Utiliser le mauvais périphérique détruira les données!

Windows

  1. Ouvrez Gestion des disques (appuyez sur Win + X, sélectionnez « Gestion des disques »)
  2. Repérez votre clé USB par sa taille (~58 Go pour une clé de 64 Go)
  3. Notez le numéro du disque (ex. : « Disque 2 »)

Ou utilisez l'invite de commandes :

diskpart
list disk
exit

Linux

lsblk

Cherchez votre clé USB par sa taille. Elle sera typiquement /dev/sdb ou /dev/sdc.

NAME   SIZE TYPE MOUNTPOINTS
sda    500G disk
├─sda1 512M part /boot/efi
└─sda2 499G part /
sdb   58.2G disk              ← Votre clé USB 64 Go
└─sdb1 58.2G part /media/user/USB

macOS

diskutil list

Cherchez votre clé USB (ex. : /dev/disk2). Vérifiez par la taille (~58 Go pour une clé de 64 Go).

Étape 3 : Sauvegarder les données existantes

Si votre clé USB contient des données importantes, copiez-les dans un endroit sûr d'abord.

Windows

Utilisez l'Explorateur de fichiers pour copier les fichiers sur votre ordinateur.

Linux

mkdir -p ~/usb-backup
cp -av /media/$USER/YOUR_USB/* ~/usb-backup/

macOS

mkdir -p ~/usb-backup
cp -av /Volumes/YOUR_USB/* ~/usb-backup/

Étape 4 : Installer Ventoy

Windows

  1. Extraire ventoy-x.x.xx-windows.zip
  2. Exécutez Ventoy2Disk.exe en tant qu'administrateur
  3. Sélectionnez votre clé USB dans le menu déroulant
  4. Cliquez sur Options → Configuration des partitions
  5. Cochez « Préserver de l'espace à la fin du disque »
  6. Entrez 47000 Mo (47 Go pour votre partition de données)
  7. Cliquez sur Installer
  8. Confirmez l'avertissement (ceci effacera le disque)
  9. Attendez la fin de l'opération

Linux

  1. Extraire et naviguer vers Ventoy :
cd ~/Downloads
tar -xzf ventoy-x.x.xx-linux.tar.gz
cd ventoy-x.x.xx
  1. Démonter la clé USB :
sudo umount /dev/sdb1 2>/dev/null
sudo umount /dev/sdb2 2>/dev/null
  1. (Optionnel) Effacer la table de partitions existante :
sudo wipefs -a /dev/sdb
  1. Installer Ventoy avec espace réservé pour la partition de données :
sudo ./Ventoy2Disk.sh -i -g -r 47000 /dev/sdb

Explication des options : - -i : Installer Ventoy - -g : Utiliser la table de partitions GPT (recommandé pour UEFI) - -r 47000 : Réserver 47000 Mo (47 Go) pour la partition de données

  1. Confirmez avec y lorsque demandé.

Exemple de sortie :

Ventoy: /dev/sdb will be formatted as GPT style.
Reserve 47000 MB disk space at the end of the disk.
Disk: /dev/sdb Size: 58.2 GB
Partition Style: GPT
Install Ventoy to /dev/sdb? (y/n) y
...
Install Ventoy to /dev/sdb successfully finished.

macOS

Ventoy n'a pas d'interface graphique native pour macOS. Utilisez l'une de ces méthodes :

Option A : Démarrez dans un environnement Linux (VM ou Live USB) et suivez les instructions Linux.

Option B : Utilisez l'interface Web de Ventoy (expérimentale) :

cd ventoy-x.x.xx
sudo ./VentoyWeb.sh

Puis ouvrez http://127.0.0.1:24680 dans votre navigateur.

Option C : Préparez la clé USB sur un ordinateur Windows ou Linux.

Étape 5 : Créer la partition EMCOMM-DATA

Après l'installation de Ventoy, vous aurez deux partitions et un bloc d'espace libre à la fin du disque :

  • sdb1 : Partition de démarrage Ventoy (~12 Go, exFAT)
  • sdb2 : Partition système EFI (32 Mo, FAT16)
  • Espace libre : ~47 Go — aucune partition n'existe encore!

Vous devez créer une nouvelle partition dans cet espace libre, puis la formater. Exécuter simplement mkfs ne fonctionnera pas — la partition doit être créée d'abord.

Windows

  1. Ouvrez Gestion des disques (appuyez sur Win + X, sélectionnez « Gestion des disques »)
  2. Trouvez votre clé USB — vous verrez les partitions Ventoy et un bloc Non alloué
  3. Clic droit sur l'espace Non alloué → Nouveau volume simple
  4. Cliquez sur Suivant, utilisez tout l'espace disponible
  5. Assignez une lettre de lecteur, cliquez sur Suivant
  6. Formatez en exFAT, nom du volume : EMCOMM-DATA
  7. Cliquez sur Terminer

Linux — GNOME Disques (Recommandé)

GNOME Disques est la façon la plus simple de créer la partition sous Linux. Il est pré-installé sur la plupart des bureaux, y compris LiaisonOS.

  1. Ouvrez Disques depuis le menu des applications (ou exécutez gnome-disks dans un terminal)
  2. Sélectionnez votre clé USB dans le panneau de gauche (identifiez-la par sa taille, ex. 64 Go)
  3. Vous verrez une barre visuelle montrant les partitions. Cherchez le grand bloc d'Espace libre à la fin (~47 Go)
  4. Cliquez sur le bloc Espace libre pour le sélectionner
  5. Cliquez sur le bouton + (plus) sous la barre de partitions
  6. Dans le dialogue qui apparaît :
  7. Taille de la partition : Utilisez tout l'espace disponible (par défaut)
  8. Nom du volume : EMCOMM-DATA
  9. Type : Sélectionnez Ext4 (recommandé) ou Autre → exFAT si vous avez besoin d'un accès Windows
  10. Cliquez sur Créer
  11. La partition est créée et formatée — c'est terminé!

Linux — Ligne de commande (Alternative)

Si vous préférez la ligne de commande, suivez ces deux étapes — d'abord créer la partition, puis la formater :

# Étape 1 : Créer la partition 3 dans l'espace libre
sudo sgdisk -n 3:0:0 -t 3:8300 -c 3:"EMCOMM-DATA" /dev/sdb

# Étape 2 : Formater en ext4 (recommandé)
sudo mkfs.ext4 -L EMCOMM-DATA /dev/sdb3

Explication des options : - sgdisk -n 3:0:0 : Créer la partition 3 avec tout l'espace restant - -t 3:8300 : Définir le type comme système de fichiers Linux - -c 3:"EMCOMM-DATA" : Définir le nom de la partition

Pour exFAT (si vous avez besoin d'un accès Windows) :

sudo mkfs.exfat -n EMCOMM-DATA /dev/sdb3

Vérifiez la disposition finale :

lsblk /dev/sdb

NAME   SIZE TYPE
sdb   58.2G disk
├─sdb1 12.3G part    ← Ventoy (EMCOMM-ISO)
├─sdb2   32M part    ← EFI
└─sdb3 45.9G part    ← EMCOMM-DATA (nouveau!)

Important : Après avoir créé la partition, définissez le propriétaire pour pouvoir y écrire :

# Monter, définir le propriétaire, puis c'est prêt
sudo mount /dev/sdb3 /mnt
sudo chown $USER:$USER /mnt
sudo umount /mnt

macOS

  1. Ouvrez Utilitaire de disque
  2. Sélectionnez votre clé USB dans la barre latérale
  3. Cliquez sur Partitionner
  4. Cliquez sur le bouton + pour ajouter une partition dans l'espace libre
  5. Nom : EMCOMM-DATA
  6. Format : ExFAT
  7. Cliquez sur Appliquer

Étape 6 : Renommer la partition Ventoy (optionnel)

Renommez « Ventoy » en « EMCOMM-ISO » pour plus de clarté :

Windows

Clic droit sur le lecteur Ventoy dans l'Explorateur → Renommer → EMCOMM-ISO

Linux

sudo umount /dev/sdb1
sudo exfatlabel /dev/sdb1 EMCOMM-ISO

macOS

Clic droit dans le Finder → Renommer

Étape 7 : Copier vos fichiers

Copier l'ISO sur la partition de démarrage

Copiez simplement votre fichier ISO sur la partition EMCOMM-ISO :

Windows / macOS : Glissez-déposez le fichier ISO avec l'Explorateur / Finder.

Linux :

cp liaisonos-debian.iso /media/$USER/EMCOMM-ISO/

Astuce : Vous pouvez garder plusieurs ISO sur la même partition. Ventoy affichera un menu pour choisir laquelle démarrer.

Copier les fichiers sur la partition de données

Copiez vos cartes, fichiers ZIM et autres ressources hors ligne sur EMCOMM-DATA.

Étape 8 : Démarrer depuis USB

  1. Insérez la clé USB dans votre ordinateur
  2. Redémarrez et accédez au menu de démarrage :
  3. Touches courantes : F12, F2, F10, ESC ou DEL (varie selon le fabricant)
  4. Mac : Maintenez la touche Option au démarrage
  5. Sélectionnez votre clé USB dans le menu de démarrage
  6. Ventoy affichera un menu des ISO disponibles
  7. Sélectionnez votre ISO LiaisonOS et appuyez sur Entrée

Paramètres BIOS/UEFI

Si le démarrage USB ne fonctionne pas, vérifiez ces paramètres :

  • Secure Boot — Désactivé
  • Boot Mode — UEFI (ou essayez Legacy/CSM)
  • Fast Boot — Désactivé
  • USB Boot — Activé

Mettre à jour votre ISO

L'un des plus grands avantages de Ventoy : pour tester une nouvelle version ISO, il suffit de :

  1. Supprimer l'ancienne ISO de EMCOMM-ISO (optionnel — vous pouvez garder les deux)
  2. Copier la nouvelle ISO sur EMCOMM-ISO
  3. Redémarrer et sélectionner la nouvelle ISO dans le menu Ventoy

Vos données sur EMCOMM-DATA restent intactes!

Mettre à jour Ventoy

Pour mettre à jour Ventoy sans perdre vos ISO ou données :

Windows

  1. Exécutez le nouveau Ventoy2Disk.exe
  2. Sélectionnez votre clé USB
  3. Cliquez sur Update (pas Install)

Linux

cd ventoy-x.x.xx
sudo ./Ventoy2Disk.sh -u /dev/sdb

Lier la partition de données à LiaisonOS

Une fois démarré dans LiaisonOS, vous pouvez créer des liens symboliques pour utiliser les données de votre partition USB :

# Monter la partition de données (si pas montée automatiquement)
sudo mount /dev/sdb3 /mnt/usb-data

# Liens symboliques vers les cartes
ln -s /mnt/usb-data/Maps/*.mbtiles ~/.local/share/liaisonos/mbtileserver/tilesets/

# Liens symboliques vers Wikipédia
ln -s /mnt/usb-data/Wikipedia/*.zim ~/wikipedia/

Note : Le nom exact du périphérique peut varier. Utilisez lsblk pour identifier la bonne partition.

Dépannage

La clé USB n'apparaît pas dans le menu de démarrage - Assurez-vous que le Secure Boot est désactivé dans les paramètres BIOS/UEFI - Essayez les modes de démarrage UEFI et Legacy - Certains systèmes plus anciens peuvent nécessiter MBR au lieu de GPT

Erreur « Logical sector size is zero » lors du renommage - La partition est en exFAT, pas FAT32 - Utilisez exfatlabel au lieu de fatlabel

L'ISO ne démarre pas - Vérifiez que l'ISO n'est pas corrompue (vérifiez le hash SHA256) - Essayez le mode « memdisk » de Ventoy : Appuyez sur F1 dans le menu Ventoy - Consultez la documentation Ventoy pour les exigences spécifiques

Partition non reconnue après création - Débranchez et rebranchez la clé USB - Sous Linux, exécutez : sudo partprobe /dev/sdb

Performance lente - Les clés USB 3.0 dans des ports USB 3.0 offrent de bien meilleures performances - Considérez une clé USB de qualité avec des vitesses soutenues - Pour les opérations critiques, l'installation permanente est recommandée

Alternative MBR pour les systèmes plus anciens :

sudo ./Ventoy2Disk.sh -i -r 47000 /dev/sdb  # Sans -g

Correction du label exFAT :

sudo exfatlabel /dev/sdb1 EMCOMM-ISO

Référence rapide

Résumé des commandes Linux

# Identifier la clé USB
lsblk

# Installer Ventoy (clé 64 Go)
sudo ./Ventoy2Disk.sh -i -g -r 47000 /dev/sdb

# Créer la partition de données
sudo sgdisk -n 3:0:0 -t 3:8300 -c 3:"EMCOMM-DATA" /dev/sdb

# Formater partition données ext4
sudo mkfs.ext4 -L EMCOMM-DATA /dev/sdb3

# Définir le propriétaire
sudo mount /dev/sdb3 /mnt && sudo chown $USER:$USER /mnt && sudo umount /mnt

# Renommer la partition Ventoy
sudo exfatlabel /dev/sdb1 EMCOMM-ISO

# Copier l'ISO
cp liaisonos.iso /media/$USER/EMCOMM-ISO/

# Mettre à jour Ventoy
sudo ./Ventoy2Disk.sh -u /dev/sdb

Disposition finale des partitions

/dev/sdb (Clé USB 64 Go)
├── sdb1  EMCOMM-ISO  ~12 Go  exFAT   [Ventoy + ISOs]
├── sdb2  (EFI)        ~32 Mo  FAT16   [Système boot]
└── sdb3  EMCOMM-DATA  ~46 Go  ext4    [Données]

Résumé : Quand utiliser quoi

  • Station EmComm principale → Installation permanente sur matériel dédié
  • BBS ou passerelle 24/7 → Installation permanente
  • Tester de nouvelles ISO → USB avec Ventoy
  • Sessions de formation → USB avec Ventoy
  • Démonstrations → USB avec Ventoy
  • Déploiement de secours → USB avec Ventoy
  • Déploiement d'urgence terrain → Les deux — USB = flexibilité, Permanent = fiabilité

Rappelez-vous : Une clé USB de démarrage bien entretenue est une excellente sauvegarde, mais un ordinateur EmComm dédié avec installation permanente reste la norme d'excellence pour le travail sérieux de communications d'urgence.
