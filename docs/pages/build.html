Building the ISO From source to bootable USB

Prerequisites

# Debian 13 (Trixie) build host
sudo apt install live-build debootstrap jq dialog

# Optional: apt cache for faster rebuilds
# (see Apt Cache section below)

Build Process

There are two build scripts:

ScriptModeDescription
setup-emcomm-iso.shInteractiveOriginal script with dialog menus (v1 + v2)
setup-emcomm-iso-v2.shInteractive or ProfileV2 script with build profile support (recommended)

Using Build Profiles (Recommended)

Build profiles are JSON files in build-profiles/ that store all build settings. This allows repeatable, headless builds without dialog prompts:

# Clone the repository
git clone https://github.com/emcomm-tools/debian-dev.git emcomm-tools
cd emcomm-tools

# Build using a profile (headless — no prompts)
./setup-emcomm-iso-v2.sh build-profiles/general-v2.json

# Or run interactively (dialog prompts)
./setup-emcomm-iso-v2.sh

{
    "language": "en",
    "arch_version": "v2",
    "overlay": "et-v2-general",
    "maps": {
        "include": false,
        "osm_regions": []

    "wallpaper": {
        "mode": "generate",
        "callsign": "emcomm-tools",
        "tagline": "Explore. Connect. Respond.",
        "color": 2,
        "center_size": 72,
        "tag_size": 50,
        "tag_offset": 170
    },
    "boot_logo": "callsign",
    "update": {
        "enabled": true,
        "url": "https://raw.githubusercontent.com/emcomm-tools/updates/main",
        "channel": "stable"
    },
    "motd": "emcomm-motd",
    "iso_type": "complete",
    "wine_prefix": "General"
}
FieldDescription
language"en" or "fr"
arch_version"v2" (active) or "v1" (frozen)
overlayOverlay to use: et-v2-general or et-v2-personal
maps.includeInclude OSM map tiles in the ISO
wallpaperWallpaper generator settings (callsign, tagline, colors)
boot_logoPlymouth boot logo style
updateOnline update channel configuration
motdTerminal banner from motd/ directory
iso_type"complete" for full ISO
wine_prefixWine prefix from wine-sources/ (VarAC, VARA)

Interactive Mode (Legacy)

Running without a profile opens dialog menus:

./setup-emcomm-iso.sh

The dialog menus let you choose:

  1. Architecture version — v1 (frozen) or v2 (active)
  2. Edition — General (no personal data) or Personal (pre-configured)
  3. Build options — Mirror, locale, etc.

Important Build Lessons

Mirror Protocol Use http:// (not https://) for the build mirror. Chroot CA certificates can cause SSL failures during lb build.
Security/Updates Repos Use --security false --updates false in lb config to avoid mixing packages from main and security repos during build. The installed system still gets security updates via the binary mirror.
Apt Cache Poisoning The save-apt-cache.sh script captures ALL .deb files from a build. If the build pulled from testing/forky, the cache has testing packages. Adding that cache as an apt source in the next build mixes testing + stable packages → dependency hell. Always verify the apt cache was created from a clean stable build before reusing.
apt-get update Returns 0 on Partial Failure If some indices fail (rate limiting, network issues), packages silently appear as "not installable" even though apt-get update returned success. Hook 0100 has retry logic and diagnostics for this.

Writing to USB

# Find your USB device
lsblk

# Write the ISO (replace /dev/sdX with your device!)
sudo dd if=live-image-amd64.hybrid.iso of=/dev/sdX bs=4M status=progress

# Or use Balena Etcher for a GUI approach
Double-check the device! dd will overwrite whatever device you point it at. Make sure /dev/sdX is your USB drive, not your system disk.
