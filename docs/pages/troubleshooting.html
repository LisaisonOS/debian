Troubleshooting Common issues and solutions

Audio Issues

Direwolf: "Audio device busy"

Symptom: Direwolf fails to start with "Audio device busy" error, especially on USB Live Run.

Cause: PulseAudio holds exclusive ALSA access on USB boot. Slower I/O on USB means PulseAudio takes longer to release the device.

Solution: Built-in retry loop in et-direwolf wrapper (5 attempts, 3s delay). No action needed. If it persists, check for other processes holding the audio device:

# Check what's using the audio device
fuser -v /dev/snd/*

# Restart PulseAudio
pulseaudio --kill && pulseaudio --start

Radio / CAT Control

rigctld not connecting to radio

Check the serial port:

# Is the radio's USB cable detected?
ls -la /dev/ttyUSB* /dev/ttyACM*

# Check udev symlink
ls -la /dev/et-radio

Check the Hamlib rig ID:

# List supported radios
rigctl -l | grep -i "icom\|yaesu\|kenwood"

# Test manually
rigctld -m 3085 -r /dev/et-radio -s 19200 &
rigctl -m 2 -r localhost:4532 f

Dashboard shows "CAT: N/A" (grey)

This is normal for audio-only radios (DigiRig Lite, FTX-1). rigctld runs in dummy mode (rig ID "1") and returns "Audio only" status. All modes work correctly — PTT via VOX or external circuit.

DigiRig Lite not detected

Check the udev rule:

cat /etc/udev/rules.d/91-et-digirig-lite.rules
# Should contain a comma before RUN+=
# Missing comma silently prevents the rule from firing

Bluetooth TNC

rfcomm0 not created after pairing

# Check Bluetooth service
systemctl status bluetooth

# Check if device is paired
bluetoothctl devices

# Create rfcomm manually (replace MAC)
sudo rfcomm bind 0 AA:BB:CC:DD:EE:FF 1

# Verify
ls -la /dev/rfcomm0

BT pairings lost on reboot (Live Run)

Cause: /var/lib/bluetooth/ exists in the read-only lower layer of OverlayFS. Writes appear to succeed but don't persist.

Solution: Use "Save & Reboot" from the persistence menu. The save script archives BT pairings as bt-pairings.tar on USB, and the restore script recreates /var/lib/bluetooth/ in the writable upper layer on next boot.

Modes

Mode won't start — precheck failure

# Check supervisor status
et-supervisor-ctl status

# Try starting with verbose output
et-supervisor-ctl start winlink-vara-hf

# Check logs
ls ~/et-supervisor-logs/
cat ~/et-supervisor-logs/<mode>-*.log

Common precheck failures:

  • No audio device — Radio USB cable not connected or not detected by udev
  • No callsign — Run et-user to set your callsign
  • No rfcomm0 — BT mode requires paired Bluetooth TNC

Terminal window closes immediately

Cause: Modes with only xfce4-terminal in the chain need the --disable-server flag. Without it, the terminal forks and the supervisor loses the PID (no STOP button).

Persistence

USB not detected for persistence

# Check mounted devices
mount | grep media

# Look for the et-config directory
find /media /run/media -name "et-config" -type d 2>/dev/null

The USB persistence partition must contain an et-config/ directory. Create it manually if needed:

mkdir -p /media/user/USB/et-config

"No USB detected" dialog on Save & Shutdown

The script shows a dialog with options to run the Setup Wizard or shutdown without saving. This is normal if the USB persistence partition isn't mounted.

Build Issues

Packages "not installable" during build

Cause: apt-get update returns 0 even on partial failure. Some package indices weren't downloaded, making packages appear unavailable.

Fix: Check network connectivity in the chroot. Hook 0100 has retry logic, but persistent network issues need manual intervention.

YAAC missing after build

Cause: unzip without -o flag prompts for overwrite confirmation in non-interactive chroot, silently failing.

Fix: Ensure hook 0170 uses unzip -o and verifies YAAC.jar exists after extraction.

Quick Diagnostics

# System info
et-system-info

# Supervisor status
et-supervisor-ctl status

# Check installed version
cat /opt/emcomm-tools/version.txt

# Check active radio
readlink /opt/emcomm-tools/conf/radios.d/active-radio.json

# Check user config
cat ~/.config/emcomm-tools/user.json

# Check if Live Run or HDD
findmnt / | grep -q overlay && echo "Live Run" || echo "HDD Install"

# List running ET processes
ps aux | grep -E "et-supervisor|rigctld|direwolf|pat|yaac|js8call" | grep -v grep
