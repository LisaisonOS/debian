Supervisor & Modes The et-supervisor daemon and JSON mode orchestration

Overview

The et-supervisor daemon is the heart of EmComm-Tools v2. It replaces 12 individual systemd services with a single Python process that manages the entire lifecycle of radio communication modes — from hardware prechecks through process startup to health monitoring.

Supervisor Components

Module File Purpose
ProcessManager process_manager.py Fork, exec, track PIDs, handle signals. States: IDLE → STARTING → RUNNING → STOPPING. Per-process log files.
ModeEngine mode_engine.py Load JSON mode configs, run prechecks (device, audio, callsign), apply config templates, orchestrate process chains.
ConfigTemplater config_templater.py Template variable substitution for INI files (JS8Call, WSJT-X), XML files (YAAC, fldigi), grid-to-latlon conversion.
DeviceChecker device_checker.py Verify hardware preconditions: udev symlinks, audio device tags, systemd services, validated callsign.
HealthMonitor health_monitor.py Periodic PID liveness and TCP port probes. Triggers crash callbacks when processes die.
IPCServer ipc_server.py Unix domain socket server. Handles JSON requests from dashboard and CLI.
GridUtils grid_utils.py Convert Maidenhead grid squares (4-6 chars) to latitude/longitude for station position.

Mode Startup Sequence

When a mode is started (via dashboard button or CLI), the supervisor follows this sequence:

  1. Stop current mode — Kill all running processes from the previous mode
  2. Load mode JSON — Read the mode configuration from conf/modes.d/
  3. Run prechecks — Verify required hardware, audio devices, and services
  4. Apply templates — Update app config files with current radio/station settings
  5. Execute pre-start actions — Run any required shell commands before starting
  6. Start process chain — Launch each process in sequence (e.g., rigctld → direwolf → yaac), waiting for each to be ready before starting the next
  7. Enable health monitoring — Begin periodic PID and port checks

CLI Usage

# List available modes
et-supervisor-ctl list

# Start a mode
et-supervisor-ctl start winlink-vara-hf

# Stop the current mode
et-supervisor-ctl stop

# Check status
et-supervisor-ctl status

# Health check
et-supervisor-ctl health

Dashboard Integration

The et-dashboard GTK panel applet communicates with the supervisor over the IPC socket. Mode buttons are dynamically generated from modes.d/ and filtered by the active radio's band capabilities. For example, if the active radio only supports VHF/UHF, HF-only modes like FT8 and JS8Call are hidden.

Status indicators show:

  • GPS — GPSD fix status (green/red)
  • CAT — rigctld connection to radio (green/red/grey for audio-only)
  • Audio — Sound card detection
  • BT TNC — Bluetooth RFCOMM device (/dev/rfcomm0)
  • UTC — Current UTC time
