Flask Web Apps Configuration UIs built with Flask and PyWebView

Overview

LiaisonOS uses lightweight Flask web applications for user-facing configuration. Each app runs on a dedicated port and opens in a native PyWebView window (GTK/WebKitGTK backend). All apps share a consistent dark theme and support bilingual EN/FR interfaces.

App Inventory

et-firstboot

First Boot Wizard — 6-step guided setup: callsign, radio selection, data transfer from USB, map downloads, drive setup.

Port 5051

et-radio

Radio selection — grid of radio cards, click to set the active radio. Shows band badges and connection type.

Port 5052

et-mode

Mode selection — choose an operating mode with modem selection dialog (1200/9600/300/VARA).

Port 5053

et-user

User configuration — callsign, Maidenhead grid, Winlink password, language preference.

Port 5054

et-chattervox-keys

Chattervox key management — view/manage secp256k1 public keys for identity verification.

Port 5055

et-tile-downloader

Map tile downloader — download OpenStreetMap .mbtiles for offline use with Navit and YAAC. Supports USB source.

Port 5056

et-radio-config

Radio configuration editor — view, edit, create, and duplicate radio JSON profiles with a form-based UI.

Port 5057

Common Patterns

Launch Modes

Every app supports three launch modes:

# Native PyWebView window (default)
python3 et-radio.py

# Open in web browser
python3 et-radio.py --browser

# Server only, no window (debugging)
python3 et-radio.py --no-browser

Bilingual Support

All apps use a TRANSLATIONS dictionary with EN/FR keys. The language is read from ~/.config/liaisonos/user.json:

TRANSLATIONS = {
    'en': {
        'title': 'Radio Selection',
        'select': 'Select',
        ...
    },
    'fr': {
        'title': 'Selection de radio',
        'select': 'Selectionnez',
        ...
    }
}

def get_language():
    config = json.load(open(ET_CONFIG_FILE))
    return config.get('language', 'en')

Dark Theme

All apps share the same CSS variable system for consistent styling:

:root {
    --bg-dark: #1a1a1a;
    --bg-card: rgba(26, 26, 26, 0.98);
    --bg-input: #2d2d2d;
    --text-primary: #e0e0e0;
    --text-secondary: #9e9e9e;
    --accent-primary: #FFA500;   /* Orange */
    --accent-danger: #f87171;    /* Red */
    --accent-success: #4ade80;   /* Green */
    --border-color: #404040;
}
WebKitGTK Gotcha PyWebView uses WebKitGTK on Linux, which renders native form elements (like <select> dropdowns) as GTK widgets. CSS cannot override their popup colors. Use custom div-based dropdowns instead of native <select> elements to ensure proper dark theme styling.

Shutdown Route

All apps implement a POST /shutdown route for graceful shutdown. PyWebView's window close event triggers this endpoint to stop the Flask server:

@app.route('/shutdown', methods=['POST'])
def shutdown():
    func = request.environ.get('werkzeug.server.shutdown')
    if func is None:
        os._exit(0)
    func()
    return 'Server shutting down...'

Desktop Menu Integration

Each app has a .desktop file in usr/share/applications/ that adds it to the XFCE application menu under the HamRadio category. Example:

[Desktop Entry]
Type=Application
Name=Radio Configuration
Name[fr]=Configuration Radio
Comment=View and edit radio profiles
Exec=/opt/liaisonos/bin/et-flask-apps/et-radio-config/et-radio-config.py
Icon=preferences-other
Categories=HamRadio;
Terminal=false
