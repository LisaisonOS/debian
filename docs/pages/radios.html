Radio Profiles JSON radio configurations and the Radio Config Editor

Overview

Each supported radio has a JSON profile in conf/radios.d/ that defines how LiaisonOS interfaces with it: CAT control (rigctld), PTT method, VARA FM settings, Bluetooth TNC configuration, and audio setup. The active radio is set via a symlink at conf/radios.d/active-radio.json.

Radio JSON Schema

Each radio profile is a JSON file with sections for different subsystems. Only relevant sections are included — a USB-only HF radio won't have a bluetooth section.

{
  "id": "icom-ic705",
  "vendor": "Icom",
  "model": "IC-705",
  "bands": ["HF", "VHF", "UHF"],

  "rigctrl": {
    "id": "3085",          // Hamlib rig model number
    "baud": "19200",       // Serial baud rate
    "ptt": "RIG",          // PTT method: RIG, RTS, DTR, NONE
    "conf": "dcd_type=NONE" // Optional hamlib --set-conf
  },

  "varafm": {              // Only for radios supporting VARA FM
    "pttPort": "COM5",     // Wine COM port
    "pttVia": "1",         // 1=RTS, 2=DTR
    "baud": "19200",
    "dtr": "0",
    "rts": "0"
  },

  "bluetooth": {           // Only for BT TNC radios
    "deviceName": "TH-D74",
    "channel": "1",        // RFCOMM channel
    "mac": ""              // User fills in their device MAC
  },

  "audio": {               // Optional ALSA setup script
    "script": "/opt/liaisonos/conf/radios.d/audio/ic705.sh"
  },

  "notes": [               // User-facing setup instructions
    "Connect USB cable to radio",
    "Set CI-V baud rate to 19200"
  ],

  "fieldNotes": [          // Developer notes
    "Tested with firmware 1.40"
  ]
}

Radio Config Editor

The et-radio-config Flask app (port 5057) provides a web UI for managing radio profiles. Users can:

  • View all radio profiles with band badges and connection type
  • Edit existing profiles (stock or custom)
  • Create new profiles from scratch
  • Duplicate and modify existing profiles
  • Delete custom profiles (the active radio cannot be deleted)
Custom Radio Storage Custom/edited radio configs are saved to ~/.config/liaisonos/radios.d/ and symlinked into the system directory. On Live Run, the persistence system saves them to USB and restores them on boot.

Selecting the Active Radio

The active radio is selected through the et-radio Flask app (port 5052) or during first boot. The selection creates a symlink:

/opt/liaisonos/conf/radios.d/active-radio.json -> icom-ic705.json

The supervisor reads this symlink at mode start to configure rigctld, Direwolf, VARA, and application templates with the correct radio parameters.
