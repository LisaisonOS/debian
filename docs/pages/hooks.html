Hook Scripts Chroot build hooks and best practices

Overview

Hook scripts run inside the chroot during lb build. They install software, configure services, set permissions, and prepare the system. Hooks are numbered for execution order and located in scripts/hooks/v2/ (v1 hooks are frozen).

Best Practices

These rules were learned through painful debugging sessions. Follow them religiously.

Always set -e Without it, failed commands are silently ignored and the build appears to succeed. You'll only discover the problem when the software is missing on the live system.
Always unzip -o Without -o, unzip prompts for overwrite confirmation in non-interactive chroot. The prompt is invisible, and unzip silently fails to extract files.
Always rm -rf before mkdir -p Stale chroot data from previous builds can leave directories populated with old files. Always clean the installation directory before creating a fresh one.
Always verify the key file after extraction After extracting an archive, verify the expected binary/jar exists. Example: 
[ -f /opt/yaac/YAAC.jar ] || { echo "YAAC.jar not found!"; exit 1; }
Never use mv for extracted directories mv can't merge into non-empty directories (from stale chroot data). Use cp -rf instead, then rm -rf the source.
Use -d flag with unzip Always specify an explicit destination directory instead of relying on cd. Easier to debug and less error-prone.
Never use [ ... ] && cmd with set -e When the test is false, the entire expression returns non-zero and set -e kills the script. Use if [ ... ]; then cmd; fi instead.
Bash ((var++)) gotcha with set -e ((var++)) returns exit code 1 when var=0 (post-increment evaluates to 0 = false in bash). Use var=$((var + 1)) instead.

Hook Template

#!/bin/bash
# =============================================================================
# NNNN-install-something.hook.chroot
# Description: Install Something for LiaisonOS
# =============================================================================

set -e

APP_NAME="Something"
INSTALL_DIR="/opt/something"
DOWNLOAD_URL="https://example.com/something.zip"

echo "=== Installing ${APP_NAME} ==="

# 1. Clean previous installation
rm -rf "${INSTALL_DIR}"
mkdir -p "${INSTALL_DIR}"

# 2. Download
cd /tmp
rm -f something.zip
wget -q "${DOWNLOAD_URL}" -O something.zip

# 3. Extract with explicit destination
unzip -o -d "${INSTALL_DIR}" something.zip

# 4. Verify key file exists
if [ ! -f "${INSTALL_DIR}/something.bin" ]; then
    echo "ERROR: something.bin not found after extraction!"
    exit 1
fi

# 5. Set permissions
chmod 755 "${INSTALL_DIR}/something.bin"

# 6. Clean up
rm -f /tmp/something.zip

echo "=== ${APP_NAME} installation complete ==="
