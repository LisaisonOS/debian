Offline Patch System Update deployed systems via USB without network connectivity

Overview

LiaisonOS is designed for field deployment where internet may not be available. The offline patch system allows updating installed systems by copying a .tar.gz patch file to a USB drive. The patch contains only changed files, SHA256 verification, and automatic rollback on failure.

Workflow

Developer Machine Target PC ───────────────── ───────── 1. Make changes to overlay 2. Run et-patch-build ├── Generate new manifest ├── Compare with old manifest ├── Stage changed files └── Create .tar.gz 3. Copy .tar.gz to USB └── /update/ folder ─────> 4. Plug in USB 5. Run et-patch-apply ├── Auto-detect patch on USB ├── Verify version match ├── SHA256 check all files ├── Backup originals ├── Apply changes └── Update version.txt

Building a Patch (Developer)

Prerequisites

  • A saved manifest from the old version (in tools/manifests/)
  • jq installed (sudo apt install jq)
  • build-config.json set to the target version

Usage

# Build patch from version 2.1.1 to current overlay
./tools/et-patch-build -o tools/manifests/manifest-2.1.1.json

# Specify output directory
./tools/et-patch-build -o tools/manifests/manifest-2.1.1.json -O ~/patches/

# Custom overlay path (for building from a different overlay)
./tools/et-patch-build -o tools/manifests/manifest-2.1.1.json \
    -d overlays/et-v2-general/overlay/opt/liaisonos

What It Does

  1. Reads old version from the old manifest and new version from build-config.json
  2. Generates a fresh manifest by scanning the overlay (SHA256 + permissions for every file)
  3. Compares old vs new manifest — classifies files as added, modified, or deleted
  4. Stages changed/added files under files/ in the patch directory
  5. Creates patch-manifest.json with the change list
  6. Includes a bootstrap copy of et-patch-apply (self-updating)
  7. Saves the new manifest to tools/manifests/ for future diffs
  8. Creates liaisonos-patch-OLD-to-NEW.tar.gz

Applying a Patch (User)

Automatic Detection

# Just plug in the USB and run:
et-patch-apply

# The script auto-scans for patches at:
#   /media/*/*/update/liaisonos-patch-*.tar.gz
#   /run/media/*/*/update/liaisonos-patch-*.tar.gz

Manual Path

# Specify patch file explicitly
et-patch-apply /path/to/liaisonos-patch-2.1.1-to-2.1.2.tar.gz

# Dry run (show what would change without applying)
et-patch-apply --dry-run

# Skip confirmation prompt
et-patch-apply --yes

Safety Features

FeatureDescription
Version check Verifies installed version matches from_version in patch manifest
SHA256 verification Every file in the patch is verified against its SHA256 hash before applying
Backup All modified/deleted files backed up to /opt/liaisonos/backup/pre-patch-VERSION/
Rollback Full automatic rollback if any verification fails during application
Dry run --dry-run flag shows all changes without modifying anything

Manifest Format

Baseline Manifest (tools/manifests/)

{
  "schema_version": "1.0",
  "distribution": "LiaisonOS Debian Edition",
  "version": "2.1.1",
  "channel": "stable",
  "release_date": "2026-03-11",
  "files": [
    {
      "path": "bin/et-supervisor",
      "version": "2.0.0",
      "sha256": "a1b2c3d4...",
      "size": 12345,
      "permissions": "755"
    },
    ...
  ]
}

Patch Manifest (inside .tar.gz)

{
  "from_version": "2.1.1",
  "to_version": "2.1.2",
  "release_date": "2026-03-13",
  "files": [
    {"path": "bin/et-dashboard", "action": "update", "sha256": "...", "size": 45678, "permissions": "755"},
    {"path": "conf/radios.d/new-radio.json", "action": "add", "sha256": "...", "size": 890, "permissions": "644"},
    {"path": "bin/old-script", "action": "delete"}
  ]
}

Building Patches Across Versions

If the current overlay has changes for version 2.1.3 but you need a patch for 2.1.1→2.1.2, use a git worktree at the 2.1.2 commit point:

# Create worktree at the 2.1.2 commit
git worktree add /tmp/et-212-patch <commit-hash>

# Set version in worktree
cd /tmp/et-212-patch
sed -i 's/"2.1.1"/"2.1.2"/' build-config.json

# Build the patch using absolute manifest path
./tools/et-patch-build -o /path/to/manifests/manifest-2.1.1.json -O /tmp

# Copy result and clean up
cp /tmp/liaisonos-patch-2.1.1-to-2.1.2.tar.gz /media/user/USB/update/
cd /home/user/liaisonos
git worktree remove --force /tmp/et-212-patch
Tip Always save manifests for released versions in tools/manifests/. These are your baseline references for building future patches.
