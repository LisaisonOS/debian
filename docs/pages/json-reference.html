Radio Profile Schema

Location: conf/radios.d/*.json

SectionFieldTypeRequiredDescription
CoreidstringYesUnique ID, matches filename (auto-generated from vendor+model)
vendorstringYesManufacturer name (Icom, Yaesu, Kenwood...)
modelstringYesModel name, may include interface type
bandsarrayYesSupported bands: ["HF"], ["VHF", "UHF"], etc.
rigctrlidstringYes*Hamlib rig model number (use rigctl -l to find). "1" = dummy/audio-only
baudstringNoSerial baud rate: 4800, 9600, 19200, 38400, 57600, 115200
pttstringNoPTT method: RIG (default), RTS, DTR, NONE
confstringNoHamlib --set-conf options (e.g., dcd_type=NONE)
pttOnlystringNo"true" if radio only supports PTT, no frequency control
primeRigbooleanNoSend initialization command to radio on connect
varafmpttPortstringNoWine COM port for PTT (COM5, COM10)
pttViastringNo"1" = RTS, "2" = DTR
baudstringNoSerial baud rate for VARA FM
dtrstringNo"0" or "1" — DTR line state
rtsstringNo"0" or "1" — RTS line state
bluetoothdeviceNamestringNoBT advertised device name (for display)
channelstringNoRFCOMM channel: "1" or "2"
macstringNoBT MAC address (user fills in their device's MAC)
audioscriptstringNoPath to ALSA setup script
notesarrayNoUser-facing setup instructions (array of strings)
fieldNotesarrayNoInternal/developer notes (array of strings)

* rigctrl.id is required if the rigctrl section is present.

Example: USB/CAT Radio (Icom IC-705)

{
  "id": "icom-ic705",
  "vendor": "Icom",
  "model": "IC-705",
  "bands": ["HF", "VHF", "UHF"],
  "rigctrl": {
    "id": "3085",
    "baud": "19200",
    "ptt": "RIG"
  },
  "varafm": {
    "pttPort": "COM5",
    "pttVia": "1",
    "baud": "19200",
    "dtr": "0",
    "rts": "0"
  },
  "notes": [
    "Connect USB cable to radio USB port",
    "Set CI-V baud rate to 19200 in radio menu"
  ]
}

Mode Configuration Schema

Location: conf/modes.d/*.json

{
  "id": "winlink-vara-hf",
  "name": {
    "en": "Winlink HF (VARA)",
    "fr": "Winlink HF (VARA)"
  },
  "description": {
    "en": "Winlink email over VARA HF modem",
    "fr": "Courriel Winlink via modem VARA HF"
  },
  "bands": ["HF"],
  "chain": ["vara-hf", "pat-http"],
  "requires": {
    "devices": ["et-radio"],
    "audio": true,
    "callsign": true
  },
  "templates": [
    {
      "type": "ini",
      "target": "~/.config/pat/config.json",
      "values": {
        "mycall": "{{callsign}}"
      }
    }
  ],
  "pre_start": [
    "et-kill-all"
  ]
}

Mode Fields

FieldTypeDescription
idstringUnique mode identifier (matches filename without .json)
nameobjectBilingual display name (en, fr)
descriptionobjectBilingual description
bandsarrayRequired bands — mode hidden if active radio doesn't support them
chainarrayOrdered list of processes to start
requiresobjectPrechecks: devices, audio, services, callsign
templatesarrayConfig file templates to apply before start
pre_startarrayShell commands to run before starting the chain
post_startarrayShell commands to run after all processes are up

User Configuration

Location: ~/.config/liaisonos/user.json

{
  "callsign": "VA2OPS",
  "grid": "FN35",
  "name": "Sylvain",
  "language": "en",
  "winlink_password": "***"
}
FieldTypeDescription
callsignstringAmateur radio callsign (uppercase)
gridstringMaidenhead grid locator (4 or 6 characters)
namestringOperator name
languagestring"en" or "fr"
winlink_passwordstringWinlink CMS password
